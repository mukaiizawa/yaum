## Project Overview

yaum (Yet Another Userscript Manager) is a Chrome Manifest V3 extension that injects user-defined JavaScript into pages on load, similar to Tampermonkey. Primary use cases: per-site behavior customization and ad removal.

**Requires Chrome 120+ with the `userScripts` API permission enabled** ("Allow user scripts" toggle in `chrome://extensions` → Details).

## No Build System

This is vanilla JavaScript — no npm, no bundler, no transpilation. Load directly as an unpacked extension via `chrome://extensions` → **Load unpacked**.

There are no lint or test commands. Manual testing is done by reloading the extension in Chrome and verifying script injection behavior on target pages.

## Architecture

### Script Injection Flow

Scripts are stored in `chrome.storage.local` (key: `scripts`, value: array of script objects). The background service worker registers all enabled scripts with the `chrome.userScripts` API. Registration is re-synced on:
- Extension install (`chrome.runtime.onInstalled`)
- Storage changes (`chrome.storage.onChanged`)
- Service worker startup

`syncUserScripts()` always does a full unregister + re-register cycle (not incremental). Calls are serialized via a `syncQueue` promise chain to prevent races.

### Regex Pattern Handling

The `userScripts` API only accepts Chrome match patterns (e.g. `*://example.com/*`), not regexes. When a script's `urlPattern` uses the `/regex/flags` syntax, `background.js:buildRegistration()` works around this by:
1. Setting `matches: ['<all_urls>']`
2. Prepending a guard `if(!new RegExp(...).test(location.href))return;` to the injected code

### File Responsibilities

- **`background.js`** — Service worker; owns `syncUserScripts()` and `buildRegistration()`; imports `storage.js` via `importScripts()`
- **`storage.js`** — CRUD for script objects in `chrome.storage.local`; shared between background and options page
- **`match.js`** — URL pattern matching (wildcard + regex); used only by the options UI
- **`options.js`** — Options page logic; reads/writes scripts via `storage.js` functions directly (no messaging)
- **`options.html` / `options.css`** — Management UI shell and styles

### Script Data Schema

```js
{
  id: string,        // generated by generateId() in storage.js
  name: string,
  urlPattern: string, // "*://example.com/*" or "/regex/flags"
  code: string,      // raw JS to inject
  enabled: boolean,
  runAt: "document_start" | "document_end" | "document_idle"
}
```

Scripts run in `world: 'MAIN'` (page's JavaScript context, not isolated).
